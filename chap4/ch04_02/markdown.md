-GCD알고리즘
GCD는 나머지 연산을 반복하며 입력 값이 빠르게 감소하므로
시간복잡도는 O(log n)이다.

피보나치 알고리즘

재귀적으로 두 번씩 호출되므로 호출 횟수가 기하급수적으로 증가한다.

T(n) = T(n-1) + T(n-2)

따라서 시간복잡도는 O(2^n)이다.

-방법
n을 5부터 증가시키며 gcd(fib(n), fib(n-1))을 계산하고,
clock() 함수를 사용하여 실행 시간을 측정하였다.

n이 증가함에 따라 실행 시간이 급격하게 증가하는 것을 확인하였다.
특히 n이 40 이상부터 실행 시간이 기하급수적으로 증가하였다.

측정 결과 n이 증가할수록 실행 시간이 급격하게 증가하였다.
특히 n이 40 이상부터 실행 시간이 약 2배씩 증가하는 경향을 보였으며,
이는 재귀적 피보나치의 시간복잡도 O(2^n)과 일치한다.
따라서 실험 결과를 통해 피보나치 알고리즘의 시간복잡도를 검증할 수 있다.

GCD 연산은 매우 빠르게 수행되며 전체 실행 시간에 큰 영향을 주지 않는다.
이는 유클리드 알고리즘의 시간복잡도 O(log n) 때문이며,
실제 측정에서도 피보나치 함수의 실행 시간이 대부분을 차지하였다.

CPU 사용량 → 거의 0%에 가까움
메모리 사용량 → 약 800KB 근처에서 거의 일정하게 유지
